# ENC-002 — Tooling і quality gates

## Мета

Налаштувати інструменти, які не дозволять реалізації розходитись із публічним контрактом і стилем проекту.

## Обсяг

- Налаштувати TypeScript strict mode.
- Додати test runner для unit та integration tests.
- Налаштувати lint і format scripts, якщо вони потрібні обраному stack.
- Додати CI workflow або локальний еквівалент quality gates, якщо CI ще недоступний.

## Критерії виконання

- `typecheck`, `test` і базова перевірка форматування/лінтингу запускаються командою з `package.json`.
- TypeScript strict mode увімкнений для production-коду.
- Quality gates не залежать від глобально встановлених інструментів.

## Межі

- Не писати повний набір поведінкових тестів у цій задачі.
- Не впроваджувати складний monorepo tooling, якщо пакет залишається окремою бібліотекою.
