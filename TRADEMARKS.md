# Trademark Policy

This trademark policy applies to the names, marks, and project identity associated with the RelicMEM project and the `@relicmem/encoding` package.

## Owner and status

For repository policy purposes, the project marks are controlled by the RelicMEM project maintainers
or by their other respective owners. This policy does not assert that any mark is registered unless
a future official RelicMEM document explicitly identifies that registered status.

The absence of a registration statement does not grant permission to use the marks in a confusing,
misleading, or endorsement-implying way.

## Project marks

The following are project marks:

- `RelicMEM`
- `@relicmem/encoding`
- the relicmem project name
- any RelicMEM logos, icons, visual marks, or distinctive project branding

These marks are used to identify the original RelicMEM project and its official packages.

You may use the project marks only as described in this policy or as otherwise permitted by applicable law.

## Allowed uses

You may use the project marks to:

- refer to the original RelicMEM project;
- refer to the official `@relicmem/encoding` package;
- describe compatibility with RelicMEM or `@relicmem/encoding`;
- identify that your work is based on or derived from RelicMEM;
- identify that your plugin, adapter, tool, article, tutorial, or integration works with RelicMEM;
- reproduce attribution notices required by the Apache License, Version 2.0.

Examples of allowed use:

```text
A plugin for RelicMEM
Compatible with @relicmem/encoding
A fork of @relicmem/encoding
Uses @relicmem/encoding internally
```

## Uses that require permission

You need explicit permission to use the project marks in a way that could imply official status, endorsement, ownership, or control by the RelicMEM project.

This includes:

- using `RelicMEM` as the main name of an unrelated product or service;
- using RelicMEM logos or visual marks in your own product branding;
- presenting a fork as the official RelicMEM project;
- publishing modified packages in a way that suggests they are official releases;
- registering domains, package names, social media accounts, or organizations that imply official RelicMEM ownership;
- using the marks in advertising that suggests endorsement by the RelicMEM project.

Examples that are not allowed without permission:

```text
Official RelicMEM Cloud
RelicMEM Enterprise Edition
RelicMEM Certified Platform
The RelicMEM Team recommends this product
```

## Forks and modified versions

You may fork, modify, and redistribute the source code under the terms of the Apache License, Version 2.0.

However, modified versions should be clearly identified as modified and should not be presented as official RelicMEM releases unless they are approved by the maintainers.

A modified distribution should use a distinct name or make the modification status clear.

Acceptable examples:

```text
ExampleCorp encoding tools, based on @relicmem/encoding
A modified fork of @relicmem/encoding
```

Avoid names that create confusion with the official package.

## Compatibility claims

You may truthfully state that your project is compatible with RelicMEM or `@relicmem/encoding`.

Compatibility claims must be accurate, limited to the versions or APIs actually tested, and must not
imply certification, endorsement, or official partnership unless such relationship has been
explicitly approved.

Acceptable examples:

```text
Compatible with @relicmem/encoding
Adapter for RelicMEM
Tested with @relicmem/encoding version 0.x
```

Not acceptable without permission:

```text
Official RelicMEM adapter
RelicMEM certified
Approved by RelicMEM
```

## Logos and visual branding

Do not use RelicMEM logos, icons, or visual marks as part of your own product identity without permission.

You may use screenshots or small references in documentation, tutorials, or reviews when the use is
accurate, non-misleading, and clearly refers to the original project.

Do not modify, recolor, combine, animate, or package RelicMEM logos, icons, or visual marks as your
own branding without explicit permission.

## No endorsement

Use of the RelicMEM name or `@relicmem/encoding` name does not imply endorsement, sponsorship, or affiliation with the RelicMEM project.

## Contact

Trademark permission requests should use a private maintainer contact published by the official
RelicMEM project. Do not open a public issue containing confidential launch plans, unreleased
branding, legal questions, or permission requests that require private review.

If no private maintainer contact is published, permission has not been granted. Use only the allowed
descriptive and compatibility references above, or wait until the project publishes an appropriate
private contact process.
